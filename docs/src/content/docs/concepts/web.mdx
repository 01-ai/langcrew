Web provides HTTP server capabilities for exposing AI agents as web services with real-time streaming communication, built on FastAPI and integrated with LangGraph's execution engine.

## What is Web?

Web in langcrew provides:
- **HTTP Server**: FastAPI-based server (AdapterServer) for agent communication
- **Streaming Communication**: Real-time agent responses via SSE (Server-Sent Events)
- **LangGraph Integration**: Direct integration with LangGraph's streaming capabilities
- **Protocol Communication**: Structured message protocols for frontend integration
- **Tool Display Management**: Rich visualization of tool calls in web interfaces
- **Frontend UI Integration**: Accompanying React-based UI components

## Core Architecture

Web is built around a minimal HTTP server that bridges AI agents with web clients:

```python
from langcrew import Agent, Crew
from langcrew.web import create_server

# Create agents
agent = Agent(
    role="Web Assistant",
    goal="Help users through web interface",
    backstory="You are a helpful web-based AI assistant"
)

# Create HTTP server
server = create_server(crew=Crew(agents=[agent]))
server.run(host="0.0.0.0", port=8000)
```

## Key Components

### AdapterServer
FastAPI-based HTTP server that handles client requests and converts them to agent executions.

**Key Features:**
- Minimal, focused design
- Built-in CORS support
- Health check endpoints
- Graceful error handling
- Custom route extensibility

```python
from langcrew.web import AdapterServer, LangGraphAdapter

# Custom server configuration
adapter = LangGraphAdapter(crew=crew)
server = AdapterServer(adapter=adapter)

# Add custom routes
@server.app.get("/custom")
async def custom_endpoint():
    return {"message": "Custom endpoint"}

server.run()
```

### LangGraphAdapter
Streaming adapter that bridges LangCrew agents with LangGraph's execution engine.

**Key Features:**
- Direct LangGraph streaming integration
- Session state management
- Event filtering and conversion
- Stop signal handling
- Tool execution tracking

```python
from langcrew.web import LangGraphAdapter

# Advanced adapter configuration
adapter = LangGraphAdapter(
    crew=crew,
    # Session management handled automatically
)

# Use with pure LangGraph
from langcrew.web import create_langgraph_server
server = create_langgraph_server(compiled_graph)
```

### Protocol Communication
Structured message protocols for consistent frontend-backend communication.

**Core Message Types:**
- `SESSION_INIT` - Session initialization
- `TEXT` / `MARKDOWN` - Text content
- `TOOL_CALL` / `TOOL_RESULT` - Tool execution
- `USER_INPUT` - HITL user input requests
- `FINISH_REASON` - Execution completion

```python
from langcrew.web.protocol import StreamMessage, MessageType

# Example message structure
message = StreamMessage(
    id="msg_123",
    role="assistant",
    type=MessageType.TEXT,
    content="Hello, how can I help you?",
    session_id="session_456"
)
```

### Tool Display Management
Manages how tool calls are visualized in web frontends with multi-language support.

```python
from langcrew.web import ToolDisplayManager

# Register tool display
ToolDisplayManager.register(
    tool_name="web_search",
    display_names={
        "en": "Searching the web",
        "zh": "正在搜索网页"
    },
    display_content_param="query"
)
```

## Communication Protocol

Web uses **SSE (Server-Sent Events)** for real-time streaming, not WebSocket:

```javascript
// Frontend integration with SSE
const eventSource = new EventSource('/api/v1/chat', {
    method: 'POST',
    headers: {
        'Content-Type': 'application/json'
    },
    body: JSON.stringify({
        message: 'Hello!',
        session_id: 'session_123'
    })
});

eventSource.onmessage = function(event) {
    const data = JSON.parse(event.data);
    console.log('Received:', data);
};
```

## Frontend UI Integration

Web includes a complete React-based frontend UI located in the `/web` directory:

**Tech Stack:**
- React 19 with modern hooks
- Ant Design (Antd) components
- Tailwind CSS for styling
- Rspack for fast bundling

**Features:**
- Real-time chat interface
- Tool call visualization
- File upload/preview support
- Multi-language support
- Responsive design

**Note:** The frontend UI has its own documentation and can be used independently.

## Server Deployment Options

### Development Server
```python
# Quick development setup
server = create_server(crew)
server.run(host="127.0.0.1", port=8000, reload=True)
```

### Production Server
```python
# Production configuration
server = create_server(crew)
server.run(
    host="0.0.0.0", 
    port=8000,
    workers=4,
    access_log=True
)
```

### Pure LangGraph Server
```python
# Direct LangGraph integration
from langgraph.graph import StateGraph

# Build your graph
compiled_graph = builder.compile(checkpointer=checkpointer)

# Create server
server = create_langgraph_server(compiled_graph)
server.run()
```

## When to Use Web

- **Interactive AI applications** requiring real-time user interaction
- **Agent-based web services** needing HTTP API endpoints
- **Streaming applications** with real-time response requirements
- **Multi-session applications** with conversation continuity
- **Tool-rich agents** requiring visual tool call feedback
- **HITL workflows** needing human approval interfaces
