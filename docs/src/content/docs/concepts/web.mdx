Web is the HTTP server component that exposes AI agents as web services, providing REST APIs, real-time streaming, and web-based interfaces for agent interaction. It transforms your AI workflows into production-ready web applications.

## What is Web?

Web in langcrew provides:
- **HTTP server**: Production-ready FastAPI server for hosting AI agents
- **LangGraph adapter**: Direct integration with LangGraph's streaming capabilities  
- **Real-time streaming**: Server-Sent Events (SSE) for live agent responses
- **Tool visualization**: Rich display of tool calls, parameters, and results
- **Frontend UI**: Complete web interface (available in `/web` directory)

## Core Architecture

Web is built on FastAPI with LangGraph integration for robust web interfaces:

```python
from langcrew.web import create_server, LangGraphAdapter

# Basic server setup
server = create_server(
    host="0.0.0.0",
    port=8000,
    cors_origins=["http://localhost:3000"]
)
```

## Key Components

### AdapterServer
Production-ready FastAPI server that hosts AI agents as HTTP services with CORS, middleware, and lifecycle management.

```python
from langcrew.web import AdapterServer, LangGraphAdapter

# Create adapter and server
adapter = LangGraphAdapter(crew=crew, enable_streaming=True)
server = AdapterServer(
    adapter=adapter,
    host="0.0.0.0",
    port=8000,
    cors_origins=["http://localhost:3000"],  # Frontend UI origin
    debug=False  # Production setting
)
```

### LangGraphAdapter
Core adapter that bridges LangCrew agents with LangGraph's streaming execution engine, enabling real-time web interactions.

```python
from langcrew.web import LangGraphAdapter
from langcrew import Agent, Crew

# Create crew and adapter
crew = Crew(agents=[agent], tasks=[task])
adapter = LangGraphAdapter(
    crew=crew,
    enable_streaming=True,      # Enable real-time streaming
    stream_tool_calls=True,     # Stream individual tool executions
    tool_display=True          # Rich tool visualization
)
```

### Protocol Types
Type definitions for HTTP communication.

| Type | Description | Usage |
|------|-------------|-------|
| `ChatRequest` | Chat message request | Client → Server |
| `StreamMessage` | Streaming response | Server → Client |
| `ExecutionInput` | Workflow execution input | Agent execution |
| `ToolResult` | Tool execution result | Tool display |
| `TaskExecutionStatus` | Task status updates | Progress tracking |

### ToolDisplayManager
Manages rich visualization of tool calls and results.

```python
from langcrew.web import ToolDisplayManager

# Configure tool display
display_manager = ToolDisplayManager(
    show_parameters=True,
    show_results=True,
    format_json=True
)
```

## Integration with langcrew

Web integrates seamlessly with all langcrew components:

- **[Agents](/concepts/agents)**: Expose agents via HTTP endpoints
- **[Tasks](/concepts/tasks)**: Execute tasks through web interfaces
- **[Crews](/concepts/crews)**: Host entire crews as web services

```python
from langcrew import Agent, Crew
from langcrew.web import create_langgraph_server

# Create agent and crew
agent = Agent(
    role="Web Assistant",
    goal="Help users through web interface",
    backstory="Expert web-based AI assistant"
)

crew = Crew(agents=[agent])

# Create web server
server = create_langgraph_server(
    crew=crew,
    host="0.0.0.0",
    port=8000
)

# Start server
server.start()
```

## Frontend UI Integration

Web comes with a complete frontend UI built with modern React technology stack:

- **Location**: Available in the `/web` directory of the langcrew repository
- **Technology**: React 19 + Antd + Tailwind CSS + Rspack
- **Features**: AgentX component, real-time streaming UI, tool visualization
- **Documentation**: The frontend has its own comprehensive documentation

The frontend UI connects seamlessly with the Web server to provide a complete web application experience.

## When to Use Web

- **Production web applications** needing AI agent backends with UI
- **API services** for mobile applications or custom frontends
- **Real-time chat interfaces** with streaming agent responses  
- **Multi-user platforms** with concurrent agent interactions
- **Enterprise applications** requiring web-based AI workflows

## API Endpoints

### Standard Endpoints

- `POST /chat` - Send chat messages to agents
- `POST /execute` - Execute specific tasks
- `GET /status` - Check server and agent status
- `WebSocket /stream` - Real-time streaming interface

### Example API Usage

```bash
# Send chat message
curl -X POST http://localhost:8000/chat \
  -H "Content-Type: application/json" \
  -d '{"message": "Hello, how can you help me?"}'

# Execute specific task
curl -X POST http://localhost:8000/execute \
  -H "Content-Type: application/json" \
  -d '{"task": "analyze_data", "input": {"data": "sample"}}'
```
